Analysis

1. Which roadmap feels more personalised?

Prompt A — Could be for a chef, a musician, or anyone. No connection to your actual life, goals, or existing skills.
Prompt B — Builds on your Python knowledge, uses your MySQL skills in Week 2, incorporates your comfort with Word/PPT. Feels written for you.

2. Which roadmap would you actually follow?

Prompt A — Hard to follow. Day 1 says "pick a skill" — you'd spend days just setting up, not learning. High abandonment risk.
Prompt B — Easy to start immediately. Day 1 task is clear: open freeCodeCamp, do HTML5. 3-hour daily blocks fit your schedule. Low friction.

3. What role did context play in improving the result?

  • Existing skills — Python + MySQL knowledge was reused, not ignored. Week 2 skips Python basics entirely.
  • Dual goal clarity — Web dev AND prompt engineering are both addressed with a logical sequence, not crammed together.
  • Time constraint — 3 hrs/day made daily tasks realistic and scoped, not overwhelming.
  • Learning style — Resources match videos + projects + reading preference (no textbook-only recommendations).
  • Level awareness — Beginner/intermediate split means Week 1 doesn't insult you with "what is a computer?"

The core lesson: Context transforms a prompt from a search query into a brief. Prompt A asked Claude to guess everything. Prompt B gave Claude the constraints needed to make real decisions — what to skip, what to prioritise, and how to connect your existing knowledge to your target goal. The quality gap isn't subtle; Prompt B is roughly 5× more actionable.
