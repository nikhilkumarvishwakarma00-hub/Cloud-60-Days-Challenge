Biggest mistakes to avoid
Using Opus for every task — it's slower and overkill for quick lookups. Reserve it for high-stakes decisions.
Vague prompts like "help me code" — always give your tech stack (Flask + MySQL + Python) and the exact error or goal.
Copying code without understanding it — ask Claude to explain every block so you actually learn, not just ship.
Using Claude only for coding — your resume needs serious work for 2026 job market. Use High effort Sonnet for career content regularly.
Starting new chats for the same project — always paste relevant context (your code, error, stack) at the top of each session for best results.
Final recommendation

If you could use only ONE model and ONE effort level: Claude Sonnet at Standard effort. It handles 80% of your daily needs — coding, learning, resume work, and research — at the right speed and depth for a BCA student in active development. When you have something important (a job application, a complex feature, interview prep), bump to High effort or switch to Opus. That one switch will make a real difference on outcomes that matter.
