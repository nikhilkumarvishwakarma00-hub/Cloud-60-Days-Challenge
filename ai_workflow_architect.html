      How to use this

      • Work through stages 1–8 in order on the left. Each stage is a complete mini-brief: what to do, which AI tool to use, exact prompts, pitfalls, and what "done" looks like.
      • Tick off tasks as you go — your progress saves automatically in this browser.

      Workflow Summary

      The whole build, compressed. Use this as your one-page mental model before you start, and as a self-check before you ship.

      The arc

      • Plan a project scoped to 1–2 weeks of evenings, not months — a finished small project beats an abandoned ambitious one.
      • Design data model and UI before touching code, so AI-assisted coding has a spec to work against instead of guessing.
      • Build backend, then frontend, wiring one vertical slice (one full feature end-to-end) before going wide.
      • Integrate and test continuously rather than at the end — catch broken contracts between frontend and backend early.
      • Deploy early and often so "it works on my machine" never becomes a launch-week surprise.
      • Document and package the project the way a recruiter actually reads it: README first, live demo link, then code.
      Recommended AI Stack

      One tool per job. This is the leanest stack that covers planning through deployment without overlap.

      JobPrimary toolWhy this one
      Scoping & specsClaude (chat)Best at structured reasoning over a fuzzy idea; turns "I want a bank app" into a real spec.
      Schema & architectureClaude (chat)Strong at reasoning about tradeoffs (SQL schema shape, route structure) before code exists.
      In-editor codingClaude Code / CursorOperates directly on your files and repo, not just a chat window — faster iteration loop.
      UI generationClaude / v0.devGood component-level HTML/CSS/JS or React generation matching a design system.
      DebuggingClaude Code / ChatGPTPaste error + surrounding code; both are strong at tracing stack traces to root cause.
      Docs & READMEClaude (chat)Strong long-form technical writing that reads like a real maintainer wrote it.
      Deployment troubleshootingChatGPT / ClaudeBoth handle platform-specific errors (Render, Netlify, Railway) well; use whichever has the freshest docs context.

      • Auto-generated changelog from commit messages using conventional commits — signals process maturity to reviewers.
      • A scheduled bot that pings you weekly with open GitHub issues on your own repo — keeps a "finished" project visibly maintained.
      • An AI code-review pass (Claude Code) wired into pre-commit hooks to catch obvious issues before push.
