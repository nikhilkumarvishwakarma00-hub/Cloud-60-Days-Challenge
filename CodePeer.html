System prompt design

The assistant is prompted as a senior software engineer running a live technical interview. It never dumps the full solution immediately — it asks the candidate to think aloud first, gives graduated hints (nudge → stronger hint → full walkthrough), and only produces the final structured report (problem restatement, approach, code, complexity analysis) once the candidate has attempted or explicitly asks to see it.

Scope is locked to DSA/interview topics; off-topic requests get politely redirected. Edge cases handled: empty/garbage input from the model, candidate asking to skip straight to the answer, candidate pasting broken code (bugs get pointed out, not silently fixed), and requests for a new problem mid-session (confirmed before switching).

UI decisions

  • Dark navy + electric blue/teal palette for a focused, distraction-free "technical" feel rather than a generic chat UI.
  • Labeled sections (Problem / Approach / Code / Complexity) render as colored tags inline so structured reports stay scannable.
  • Difficulty/topic/language locked in before the chat opens, mirroring how a real interview is scoped upfront.
  • Code blocks use JetBrains Mono with syntax-neutral styling to stay readable across languages.

Future extensions

  • Tools: a code execution tool so the assistant can actually run the candidate's code and report real output/errors instead of reasoning about it statically.
  • Memory: persist topic-level performance across sessions (localStorage or a backend) to recommend which weak areas to revisit.
  • Multi-step workflows: a full mock-interview mode with a timer, a closing "interviewer feedback" summary, and a difficulty ramp that adapts based on how the candidate performed on the previous problem.
